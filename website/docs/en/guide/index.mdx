# Guide

vitest-agent is a Vitest plugin, CLI, and MCP server family that turns test
runs into structured, token-efficient signal for AI coding agents and enforces
a disciplined test-driven development workflow. Instead of thousands of lines
of console noise, agents receive a compact classified final frame — counts,
classifications, and a direct pointer to the right MCP tool to act on the
result.

## The two-layer model

vitest-agent ships as two distinct layers that stack on top of each other.

**Layer 1: the npm plugin (everyone installs this)**

`vitest-agent-plugin` is the data and collection foundation. Add it to your
`vitest.config.ts` and it handles environment detection, per-executor console
modes, failure classification, coverage policy, and persistence to a local
SQLite database. The CLI and MCP server are pulled in automatically as
required peer dependencies.

**Layer 2: the Claude Code marketplace plugin (optional)**

`vitest-agent@spencerbeggs` is the agent-behavior layer. It auto-registers the
MCP server with Claude Code, adds lifecycle hooks that inject test context at
session start, ships a library of TDD skills, and provides a `tdd-task`
orchestrator subagent driven by the `/tdd` command. If you use Claude Code,
this companion plugin is how the collected data becomes active agent behavior.

You install the npm plugin first. If you use Claude Code, you then add the
marketplace plugin as a companion.

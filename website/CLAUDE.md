# vitest-agent docs site (`docs` package)

RSPress 2.0 documentation site for the whole vitest-agent family, deployed to <https://vitest-agent.dev> via Cloudflare Pages. Private workspace (npm name `docs`); never published and versions independently of the runtime packages.

- Source root: `docs/`, locale-scoped under `docs/en/` (`root: "docs"`, `lang: "en"` in `rspress.config.ts`). Single-locale (English) today. Homepage is `docs/en/index.mdx`.
- Top nav is Guide and Packages, wired in `docs/en/_nav.json` (Packages is a dropdown of the seven packages). Per-section sidebar ordering lives in each directory's `_meta.json`.
- `docs/en/guide/` is the learning spine. Each package owns `docs/en/<short>/` (`plugin`, `sdk`, `mcp`, `cli`, `reporter`, `ui`, `sidecar`) with an Overview `index.mdx`, hand-written deep-dives, and a generated `api/` subtree. `docs/en/packages/` is the ecosystem map.
- API pages are generated by the published `rspress-plugin-api-extractor`, configured as `ApiExtractorPlugin.api.fromModelsDir("./lib/models")`. Each package's `rslib.config.ts` `apiModel.localPaths` copies its model into `website/lib/models/<short>/` during `build:prod`, so the site's `build` task `dependsOn` every package's `build:prod` (see `turbo.json`).
- `website/api-docs-snapshot.db` is committed as the generation source of truth so CI reproduces API pages deterministically. The copied models (`lib/models/*`), generated pages (`docs/en/*/api/`), and the snapshot's `-shm`/`-wal` sidecars are gitignored (rules consolidated in the root `.gitignore`).
- Build: `pnpm --filter docs build`. Dev: `pnpm --filter docs dev` (port 4173, override with `DEV_PORT`). Output bundle is `website/dist/`.
- The repo-root `docs/*.md` files are superseded by this site; new user-facing prose belongs here.
